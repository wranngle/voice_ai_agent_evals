### 1.0.0-dev (in progress on `feat/v1.0-bun-package`)

Rename to `@wranngle/voice-evals` and rebuild as a published Bun package with audio-native voice-AI evals, closed-loop remediation, and dynamic test detection. Phase tracker:

- **Phase 0 — Package shell** (current). Rename `lib/` → `src/`. Add ESM+CJS dual build via `bun build` + `tsc --emitDeclarationOnly`. Wire `exports`, `bin`, `files`, `engines`, `publishConfig`, `peerDependencies` on `@elevenlabs/elevenlabs-js`. Move `arktype` to runtime `dependencies`. Stub `scripts/postinstall.mjs` for the Phase 5 Python sidecar. README + badges updated.
- **Phase 1 — ElevenLabs wrapper.** Lands `src/wrapper/` with: `createVoiceEvalsClient({apiKey | client, modelRankings?})` factory; `agents.list()` / `agents.get(id)` that parse `[PHASE]` prefixes per AGENTS.md; `governance.{parseAgentName, enforceMutation, assertModelAllowed, isPhase, GovernanceError}` pure helpers; `tools.{cleanProperty, cleanTools, hasMutualExclusionViolation}` pure helpers that strip the API's mutually-exclusive fields (`is_system_provided` / `dynamic_variable` / `constant_value` / `enum`) before PATCH and synthesize replacement descriptions; `webhooks.verify` re-export of the HMAC verifier. Subpath export `@wranngle/voice-evals/wrapper`. `config/model-rankings.json` ships with the AGENTS.md banned list (`gpt-4o-mini`, `gpt-5-mini`, `gemini-2.0-flash-001`) and default `gemini-3-flash-preview`. SDK escape hatch via `client.raw`. New vitest `wrapper` project with 46 tests covering all of the above; CI workflow updated.
- **Phase 2 — Scoring engine** (MVP slice). Inspect-AI-shaped composable scoring under `src/scoring/`: `Task = (dataset, caller, scorer)` primitive; `compose(...scorers)`, `weighted(weight, scorer)`, `aggregate(dimensions) -> RunOutcome` composer; Promptfoo-style assertions DSL (`contains`, `regex`, `equals`, `not`, `notAsync`, `llmRubric` with judge-callback) including `not-*` negation that preserves error / skipped states; audio-native scorers — `parseWav` (16/24/32-bit PCM, mono + stereo deinterleave), `rmsEnvelope` (50ms windows), `detectSpeechSegments` (energy-threshold VAD with smoothing), `detectBargeIn` (two-stream stereo barge-in), plus dimension-emitting `scoreVoiceActivity` and `scoreBargeIn`. Subpath export `@wranngle/voice-evals/scoring`. 44 new tests in a `scoring` vitest project with synthesized WAVs (no binary fixtures committed). Migration from `src/testing/runners/scenario-runner.ts` onto the new primitives is deferred to Phase 2.x; judges (`g-eval`, `arena`, `dag`, `rubric`, `lynx`) and rolling-p95 latency scorers also deferred to Phase 2.x.
- **Phase 3 — LLM data layer** (MVP slice). Three ingestion paths under `src/ingestion/`: (1) `importPostCallWebhook(payload, opts)` — deterministic converter from an ElevenLabs `post_call_transcription` webhook payload to `TestCase[]`, one TC per `evaluation_criteria_result` (or one placeholder when no analysis is present). Pure function; injectable clock. (2) `proposeTestCases(transcript, {llm})` — TestChain Proposer: LLM callback reads a transcript and emits `ProposedTestCase[]` with `suggested_id`, `simulated_user.first_message`, `draft_assertions`, optional `persona`. Tolerates `\`\`\`json\`\`\`` code-fences and `{cases:[...]}` envelopes; drops items that fail schema validation. (3) `persona-generator` — 5 `CANONICAL_PERSONAS` (polite-elderly, frustrated-rusher, esl-non-native, confused-meanderer, hostile-skeptic) with traits (`pace_wpm`, `interruption_tendency`, `disfluency_rate`, `frustration_slope`, `accent`) and `buildPersonaSystemPrompt(persona, intent)` for simulated-user prompting. Subpath export `@wranngle/voice-evals/ingestion`. New vitest `ingestion-llm` project (named to avoid collision with the legacy `ingestion` project for vitest file parsing); CI workflow updated. 17 new tests. Designer step (proposal → structured assertion specs) and adversarial bridge to PyRIT are deferred to Phase 3.x / Phase 5.
- **Phase 4 — Regression + baseline** (MVP slice). `src/regression/` ships versioned `BaselineSnapshot` capture / save / load (file-based JSON under `baselines/<name>.json`, path-traversal-safe slug) and the Braintrust-shaped `diffAgainstBaseline(current, baseline)` returning `{scores: {baseline, current, diff}, perTest, improvements, regressions, unchanged, newTests, droppedTests, dimensionDiffs}`. Status flips (passed→failed or vice versa) always land in improvements / regressions regardless of magnitude; numeric deltas below `unchangedThreshold` (default 0.02) land in `unchanged`. Pure function with no I/O; CI gates on `result.regressions.length`. Subpath export `@wranngle/voice-evals/regression`. New vitest `regression` project with 11 tests. Drift detection (statistical-significance gates via permutation tests) and the dedicated trace-to-test convenience wrapper are deferred to Phase 4.x; the deterministic trace-to-test path is already available via `importPostCallWebhook` from Phase 3.
- **Phase 5 — Closed-loop remediation** (MVP slice + Python sidecar contract). `src/remediation/` exposes: (1) `proposeFix({llm, agentConfig, failures, context?}) -> FixProposal[]` — single-shot LLM proposer over 8 targets (`system_prompt`, `tool_description`, `first_message`, `voice_stability`, `voice_similarity_boost`, `voice_speed`, `temperature`, `turn_eagerness`) with defensive sanitization (numeric clipping, enum forcing) so a hallucinated LLM proposal can't burn an agent. (2) `applyFix({client, agentId, fix, governance?, dryRun?}) -> ApplyFixResult` — translates the proposal into a `conversationConfig` patch, enforces the `[DEV]`-only mutation gate via the wrapper, and PATCHes the SDK (camelCase per Fern bindings). Dry-run mode returns the would-be patch without sending. (3) `polishLoop({client, agentId, evaluate, llm, maxIterations?, patience?, dryRun?, governance?}) -> PolishLoopResult` — iterates evaluate → propose → apply → evaluate with plateau detection (`patience` consecutive no-improvement iterations), max-iteration cap, all-passing short-circuit, and `no_proposal` early exit. Every step recorded in `history[]` for audit. (4) `gepa-bridge.ts` ships the Phase 5.x contract: `isGepaAvailable()`, `runGepaOptimization(input)`, `GepaUnavailableError`. Today the bridge throws with installation instructions; the Python sidecar venv install + `gepa_run.py` JSON-IO protocol lands in Phase 5.x (PyRIT adversarial sidecar uses the same install path). Subpath export `@wranngle/voice-evals/remediation`. New vitest `remediation` project with 16 tests; CI workflow updated.
- **Phase 6 — CLI + docs + CI + release** (in progress). `src/cli.ts` extended with a `voice-evals doctor` subcommand that prints the Python sidecar status (cache path, venv binary, bridge script, availability). README rewritten for v1.0 positioning: quickstart code that touches every namespace (wrapper, scoring, ingestion, regression, remediation), capability table vs. Hamming / Coval / Vapi / ElevenLabs sim, subpath-exports reference, and an updated phase tracker. `examples/quickstart.ts` ships a runnable end-to-end demo using synthesized data. Still pending: the npm publish (`bun publish` once the user is ready) and the GitHub repo rename to `wranngle/voice-evals` — both are user-authorized actions, not autonomous. Matrix CI on Node 20/22 + Bun, the docs-site rewrite, the full split of `src/cli/commands/`, and the dropped legacy `src/extraction/` rename are deferred to Phase 6.x.

## Versioning

- **MAJOR** — breaking changes to the runner contract, scenario YAML schema, or stored-test data shape
- **MINOR** — new runner types, new scoring axes, additional CLI commands
- **PATCH** — bug fixes, doc clarifications, dependency bumps

Deprecations will be announced here with at least 90 days notice before removal.
